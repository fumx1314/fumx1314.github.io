简介

SSRF (Server-Side Request Forgery,服务器端请求伪造)是一种由攻击者构造请求，由服务端发起请求的安全漏洞。一般情况下，SSRF攻击的目标是外网无法访问的内部系统(正因为请求是由服务端发起的，所以服务端能请求到与自身相连而与外网隔离的内部系统)。

即利用服务器的权限去请求一些其他的非法内容

SSRF全详解-CSDN博客

file_get_contents()

可以读取本地文件或远程资源的内容，并返回字符串。

支持多种协议（如 http://、https://、ftp://、file:// 等）

file_put_content函数把一个字符串写入文件中。

fsockopen()

是 PHP 用于创建网络连接的函数，主要用于：

  • 建立 TCP/UDP 连接

  • 访问远程服务器

  • 进行 Socket 通信

    语法

参数解析

参数 说明
$hostname 目标主机（可以是 IP 或域名）。支持 tcp://、udp://、ssl:// 等协议。
$port 目标端口，如 HTTP（80）、HTTPS（443）。
$errno 可选，用于存储错误代码。
$errstr 可选，用于存储错误信息。
$timeout 可选，设置超时时间（秒），默认使用 php.ini 的 default_socket_timeout 设置。

curl_exec()

是 PHP cURL 扩展中的一个函数，用于执行 cURL 请求并获取服务器响应。

curl_exec() 作用：

  • 发送 HTTP 请求到 http://example.com
  • 获取服务器响应数据
  • 存储到 $response 变量中

常用伪协议

  • file协议： 在有回显的情况下，利用 file 协议可以读取任意文件的内容
  • dict协议：泄露安装软件版本信息，查看端口，操作内网redis服务等
  • gopher协议：gopher支持发出GET、POST请求。可以先截获get请求包和post请求包，再构造成符合gopher协议的请求。gopher协议是ssrf利用中一个最强大的协议(俗称万能协议)。可用于反弹shell
  • http/s协议：探测内网主机存活

gopher

gopher协议在ssrf的利用中一般用来攻击redis，mysql，fastcgi，smtp等服务。

基本格式：

gopher://<主机>:<端口>/<类型><路径>

<主机>：Gopher 服务器地址（例如 gopher.example.com）。

<端口>：Gopher 服务器端口（通常是 70）。

<类型>：Gopher 资源类型（0=文本文件，1=目录，9=二进制文件等）。

<路径>：文件或目录的路径。

注意：

  • gopher协议数据流中，url编码使用%0d%0a替换字符串中的回车换行
  • 数据流末尾使用%0d%0a代表消息结束
  • curl_exec()造成的SSRF，gopher协议需要使用二次URLEncode;而file_get_contents()造成的SSRF，gopher协议就不用进行二次URLEncode；

ftp

作用：

探测内网存活主机（端口扫描）

枚举 FTP 服务器（是否允许匿名访问）

获取文件列表或读取文件

配合 Gopher、DICT 等协议构造恶意请求

1、FTP 用于探测内网存活主机

由于 FTP 服务器通常运行在 端口 21，可以通过 SSRF 来判断目标是否开放 FTP 服务。

📌 示例

利用方式

  • 如果服务器返回响应，说明目标 192.168.1.1 开放了 21 端口，可能存在 FTP 服务。
  • 如果请求超时，可能目标主机不存在，或者防火墙阻止了访问。

2、利用 FTP 进行文件读取

FTP 支持 LIST 命令，部分 FTP 服务器允许匿名访问（Anonymous Login），可以通过 SSRF 来列举 FTP 服务器上的文件。

📌 示例

  • 如果返回文件列表，说明 FTP 服务器允许匿名访问。
  • 可以进一步尝试读取文件（如 ftp://anonymous:anonymous@192.168.1.100/flag.txt）。

3、利用 FTP 读取配置文件

有些 FTP 服务器会存储敏感文件，如数据库配置 (config.php)、日志文件 (log.txt) 等。如果 FTP 允许匿名访问，就可以通过 SSRF 读取这些文件。

📌 示例

  • 如果成功，可能会返回 /etc/passwd 文件内容，包含系统用户信息。
