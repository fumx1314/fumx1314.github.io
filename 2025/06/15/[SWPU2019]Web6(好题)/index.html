攻击原理：

'1'='1' → 恒真条件，绕过用户名检查。

GROUP BY passwd WITH ROLLUP → 对 passwd 分组，并生成一个汇总行（ROLLUP 会额外计算聚合值）。

HAVING passwd IS NULL → 筛选 passwd 为 NULL 的行（ROLLUP 生成的汇总行 passwd 是 NULL）。

# → 注释掉后续的 AND passwd='$pass'，使密码检查失效。

这样查询会返回我们的密码为null，而我们这时输入的密码也输入为空即可

SOAP 是基于 XML 的简易协议，可使应用程序在 HTTP 之上进行信息交换。

根据前面method需要admin和127.0.0.1来访问get_flag方法，我们可以用SoapClient来调用get_flag方法，SoapClient实际上并没有这个方法但是_call方法会帮助我们调用SoapServer中的get_flag方法

SoapClient中调用SoapServer中的方法

call_user_func函数有调用成员方法的作用，所以我们令$d->flag='get_flag'，$d->b='call_user_func'，然后将$a数组中第一个成员设为SoapClient类即可，这里可以利用php序列化的属性，在se.php文件中有ini_set('session.serialize_handler', 'php');，这里将php序列化处理器设置为php，只要我们在session中写入含有|的字符串，php会自动将|前的字符串作为键，其后的字符串作为值反序列化

（1）第一阶段：利用 PHP_SESSION_UPLOAD_PROGRESS 反序列化

  • PHP_SESSION_UPLOAD_PROGRESS：
    • PHP 在上传文件时会自动记录进度，并存储在 $_SESSION 中。
    • 如果攻击者控制 PHP_SESSION_UPLOAD_PROGRESS，可以注入恶意数据。
  • Payload：
    • |O:10:"SoapClient":5:{...} 是一个 PHP 反序列化字符串，构造了一个 SoapClient 对象。
    • SoapClient 可以发起 HTTP 请求（SSRF），用于后续攻击。

（2）SoapClient 反序列化 SSRF

  • 作用：
    • 反序列化后，SoapClient 会向 http://127.0.0.1/interface.php 发起请求。
    • _user_agent 可控，可以伪造 HTTP 头（如 X-Forwarded-For、Cookie）。
    • 可能用于 绕过 IP 限制 或 伪造身份。

（3）第二阶段：触发 se.php 反序列化

  • Payload：

    • 这是一个 PHP 对象注入（POP链），利用目标代码中的 unserialize() 执行任意操作。

    • 关键部分：

      • 调用 call_user_func("get_flag")，可能触发目标系统执行 get_flag 函数，返回敏感信息（如 flag）。
