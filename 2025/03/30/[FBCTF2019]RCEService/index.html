刷题

题目界面如上

它说需要我们传入一个json格式的命令

我们传入

{"cmd":"ls"}

回显出来了index.php

但当我们传入

{"cmd":"ls /"}

发现/被过滤了

经过多次尝试，发现好多符号都被过滤了

没办法了，看题解

题解说这题是有源码的…我没找到…题目的问题

题目源码：

解释代码

$cmd = json_decode($json, true)['cmd'];
json_decode(): 这个函数用于将JSON格式的字符串解码为PHP变量,参数 $json 是要被解码的 JSON 字符串,格式类似 {“cmd”: “ls”}

json_decode第二个参数被设置为true,意味着json_decode函数将返回一个关联数组(array)而不是一个对象(object),把json_decode函数返回的关联数组中的’cmd’键对应的值赋值给变量$command

正则：^.(捕获组).$

解法一: PCRE回溯机制绕过正则表达式

这题其实用post传cmd也是可以的

根目录下没找到flag

我们用find找一下

payload = '{"cmd":"find / -name flag*", "abc":"'+'a'*1000000+'"}'

发现还是没有

按道理来说不应该，我们试着find index.php，这个文件肯定是有的，如果还是没有回显，那就是有问题了

发现还是没有回显

那么应该是没有成功执行find 命令(题解说的)

最可疑的就是

putenv('PATH=/home/rceservice/jail');

find命令在我们自己的系统里是在**/bin/** 和 /usr/bin/ 目录下都存在 这里只有/usr/bin/find 路径可以使用

所以payload = '{"cmd":"/usr/bin/find / -name flag", "abc":"'+'a'*1000000+'"}

成功回显出了/home/rceservice/flag路径

那现在就是要读取flag了，但cat命令也不能直接使用，cat在我们的系统里和find命令一样在 /bin/ 和 /usr/bin/ 目录下都存在

但这里只能用/bin/cat

payload = '{"cmd":"/bin/cat /home/rceservice/flag", "abc":"'+'a'*1000000+'"}'

读到了flag

解法二: %0a 绕过

讲解

如果在字符串最末尾的 } 之前加上一个%0a 变成

此时.* 匹配 { 而\x00-\x1F 匹配了第一个%0a 但是最后的 .* 不能匹配换行符,因此也匹配不到换行后的 } 所以不能匹配到完整字符串,返回值为空,完成正则绕过

简记为：

^xxx$这个格式的绕过一般采用%0a

preg_match()函数只能匹配第一行数据，可以使用换行符%0a绕过

pyload: ?cmd={%0A"cmd":"/bin/cat /home/rceservice/flag"%0A}

得到flag

当然，前后多加几个%0A也没事
